# AI 디자인 제작 작업 안내

이 저장소는 업체가 바뀌어도 같은 방식으로 디자인 시안을 만들 수 있도록 정리한 **AI 디자인 템플릿 작업 폴더**입니다. 공통 스타일과 제작 규칙은 `templates/`에 두고, 업체별 정보와 결과물은 `clients/`에 분리합니다.

## 폴더 역할

- `templates/`: 여러 업체에 반복해서 사용할 공통 디자인 템플릿과 제작 규칙을 보관합니다.
- `clients/`: 업체별 브랜드 정보, 제품 요청서, 참고 이미지, 최종 결과물을 보관합니다.
- `assets/shared/`: 여러 업체에 함께 쓸 수 있는 공통 이미지, 패턴, 목업 자료를 보관합니다.
- `output/`: 임시 출력 또는 전체 프로젝트 공통 결과물을 둘 수 있습니다. 업체별 최종 결과물은 되도록 `clients/<업체명>/output/`에 둡니다.

## 기본 작업 흐름

1. 새 업체가 생기면 `clients/<업체명>/` 폴더를 만듭니다.
2. 업체별 `business-context.md`, `design-brief.md`, `data.json`을 작성합니다.
3. 작업 종류에 맞는 템플릿을 `templates/`에서 고릅니다.
4. 참고 이미지와 로고는 `clients/<업체명>/assets/`에 모읍니다.
5. 최종 시안은 `clients/<업체명>/output/`에 저장합니다.

## 현재 준비된 공통 템플릿

- `templates/henna-label/`: 헤나 염색약 라벨 디자인용 공통 템플릿입니다.

## 현재 준비된 업체

- `clients/dada/`: DADA 미용 브랜드의 헤나 염색약 라벨 작업 자료입니다.

## 작업 원칙

- 공통 템플릿 파일에는 특정 업체만의 문구나 이미지가 들어가지 않게 합니다.
- 업체별 파일에는 해당 업체의 브랜드명, 제품명, 색상명, 용량, 참고 이미지를 정리합니다.
- 참고 이미지는 방향을 잡기 위한 자료로만 사용하고 그대로 따라 하지 않습니다.
- 실제 인쇄 전에는 라벨 크기, 필수 표기 문구, 인증 문구 사용 가능 여부를 반드시 확인합니다.
- 초보자도 이해할 수 있는 쉬운 말로 문서를 작성합니다.

## 멈춤 기준

새 업체용 시안을 만들기 전에는 업체별 정보가 충분히 정리됐는지 먼저 확인합니다. 확정되지 않은 제품 정보는 임시값으로 표시하고, 최종 인쇄용 파일이라고 단정하지 않습니다.
